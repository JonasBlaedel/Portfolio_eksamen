I Tema 2 fik vi en grundlæggende forståelse for og gjorde brug af de mest anvendte metoder, begreber og programmer i en multimediedesigners “værktøjskasse”.

Vi blev introduceret til nogle grundlæggende faglige begreber, når man som multimediedesigner arbejder med diverse brugergrænseflader, digitale former for kommunikation og responsive designs.
Dette indebærer en generel forståelse for opsætning af websider, at arbejde i grafik og billedbehandlingsprogrammer samt at opsætte tekst og billeder til mockups, stiletyles, wireframes.
I den forbindelse lærte vi også om metoder til digitalt design, som diverse design konventioner - stilarter, genrer, stilperioder mm. - forståelse og brug af forskellige gestalt principper og ikke mindst farveharmonier.

I dette projekt skulle jeg udvikle et responsivt website bestående af 5 sider ud fra afleveret tekst, billeder, wireframe og layoutdiagram. Udseendet skulle følge wireframet, og det strukturelle layout for den enkelte side skulle opsættes efter layoutdiagrammet med en detaljeret opsætning af de strukturelle HTML elementer.
Websitet udførtes ved hjælp af CSS grid, flex og ikke mindst media queries - essentielt for det responsive aspekt. Samtidig skulle websitet gøre brug af forskellige farver, have to forskellige fonte samt inkludere meta-tagget; "meta name="robots" content="noindex"" i head-elementet, så robotter ikke automatisk inlkuderede det i søgemaskiner.
Desuden skulle websitets HTML validere uden fejl.

Den responsive hjemmeside skulle tage udgangspunkt i det tidligere mobilsite. Jeg løste opgaven ved først at få et overblik over det udleverede wireframe og layoutdigram. Gennem wireframet kunne jeg se det visuelle design opnået med CSS, og via Layout Diagrammet vidste jeg hvordan jeg skulle opbygge HTML strukturen.

Med udgangspunkt i ovenstående, startede jeg med at oprette de fem forskellige html-sider og html strukturens elementer med ‘parents’ og ‘children’ og navngav alle classes.
Jeg tog så udgangspunkt i wireframen. Her var beskrevet hvilke tekst- og billedelementer der skulle befinde sig i diverse containers - såsom header-tags, paragraphs images osv.
Nu kom det til at style websitet. Siden HTML kun er den grundlæggende struktur og konstruktion af websitet, skulle jeg bruge CSS til at style. CSS står for Cascading Style Sheet, og er et deklaratorisk sprog, hvilket betyder, at man skaber deklarationer for diverse elementer - deklarationen der står nederst er den gældende for det enkelte element. Helt grundlæggende fungerer CSS sådan, at man styler en selector, som befinder sig i dit HTML dokument. Ved at give din selector en property med en dertilhørende value, skaber du et andet udseende i modsætnings til din browser standard deklaration.

Altså stylede jeg HTML dokumentet og igen tog jeg udgangspunkt i wireframet så alle container var placeret på deres rigtige pladser.
I header elementet brugte jeg CSS grid i fem ens fractions og stylede mine grid-items med justify-items: center; så navigationen stod midt for. I main brugte jeg også CSS grid. Jeg stylede class’ne efter deres navne ex. .grid_3-5. Altså skulle denne class have en property og value med display:grid; og grid-template-columns: 3fr 5fr;. Hvis ikke html strukturen forhindrede elementer i at sidde i den rigtige rækkefølge, måtte jeg bruge grid-row og grid-columns til at manipluere deres positioner.

For at hjemmesiden blev responsiv, brugte jeg media queries med en betingelse, at CSS grid stylingen kun måtte gælde, hvis browser vidden var over 800px. På den måde forsvandt alt styling under 800px, og strukturen vendte tilbage til den, der var opsat til mobilversionen er hjemmesiden.
